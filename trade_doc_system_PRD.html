01
产品背景与定位
Background & Positioning

外贸企业在日常运营中会产生大量高价值单据，包括报关单、商业发票、装箱单、银行水单、提单等。这些文件目前多以纸质或散乱电子文件形式存在，查询困难、数据沉睡、无法被二次利用。

本系统定位为外贸企业文件数字化基础设施，以电子化归档为起点，逐步构建结构化数据资产，最终通过大数据技术清洗整理，持续训练企业私有业务大模型，形成数据飞轮效应。

02
核心痛点
Problem Statement
📦
文件散乱，查询困难
纸质单据与电子文件混存，按订单号、客户名查找耗时，跨部门协作时文件传递效率低。
💤
数据沉睡，无法复用
报关单、发票中包含大量结构化数据（金额、HS编码、重量等），但以图片或PDF形式封存，无法统计分析。
⚠️
人工核对，错误率高
发票与报关单金额、箱单与提单数量是否一致，依赖人工逐一比对，费时且易出错，合规风险大。
🧠
决策缺乏数据支撑
市场趋势、客户贡献、商品结构等分析需手动整理报表，无法及时响应业务变化，大模型能力无从建立。

05
核心功能模块详述
Feature Specifications

5.1 文件采集与录入

支持多种文件采集方式，满足不同场景需求：

采集方式说明支持格式优先级
手动上传用户在系统内直接拖拽或点击上传文件PDF、JPG、PNG、Word、ExcelP0
邮件自动采集绑定企业邮箱，自动识别附件并入库邮件附件各格式P1
API 对接与 ERP / 海关系统打通，自动同步文件JSON / XMLP2
扫描仪对接企业局域网扫描仪直接推送至系统PDF / TIFFP2

5.2 智能字段提取（OCR + NLP）

上传后自动触发识别流水线，关键字段无需人工录入：

目标识别率：OCR 文字识别 ≥ 98%，关键字段自动提取率 ≥ 94%，低置信度字段自动推送人工核对队列，确保数据质量。
以报关单为例字段类型提取方式备注
报关单号String● 自动提取正则 + 模板匹配
申报日期Date● 自动提取标准化为 YYYY-MM-DD
HS 编码String● 自动提取8/10位格式校验
成交金额Decimal● 自动提取含币种识别
货物名称String● 自动提取中英文均支持
贸易方式Enum⚠ 人工确认置信度低时推送核对
监管条件String⚠ 人工确认可选字段，按需录入

5.3 文件管理与检索

电子化归档后的文件支持多维度管理：

多维检索
按文件类型、日期范围、客户/供应商名称、关联订单号、金额区间全文检索，支持模糊匹配。
跨文件关联
同一票货的发票、装箱单、提单、报关单自动关联为一组，点击任意文件可查看全套单据。
版本管理
支持同一文件的修订版本留存，可比对任意两个版本的字段差异，满足审计需求。
权限控制
按角色（管理员/业务员/财务/只读）分配文件类型访问权限，支持部门级数据隔离。详见第06章。

5.4 文件预览与标注

归档文件支持在线预览和可视化标注，减少下载操作，提升核对效率：

在线预览
支持 PDF、JPG、PNG、Word、Excel 等格式在线预览，无需下载即可查看文件内容，加载时间 < 2秒。
OCR 标注叠加
在原件上以半透明高亮框标注 OCR 提取位置，点击高亮区域可直接跳转到对应字段编辑。
人工圈注修正
核对人员可在文件预览上用画笔圈注有疑问的区域，添加批注说明，批注随文件版本保存。
双栏对照视图
左侧展示原件图像，右侧展示提取后的结构化字段表单，支持逐字段定位对照。

5.5 批量操作

针对外贸业务中频繁的批量处理需求：

批量操作说明限制优先级
批量上传一次拖拽多个文件，系统自动识别每个文件类型并分别进入 OCR 流水线单次最多 50 个文件P0
批量导出按筛选条件（日期、类型、客户、订单号）勾选文件，打包下载原件或导出结构化数据 Excel单次最大 500MBP0
批量打印选择一票货物的全套单据一键打印，自动按报关单→发票→箱单→提单排序—P1
批量状态变更管理员可对筛选后的多个文件批量执行归档、作废等操作需二次确认P1
批量核对同一票货的多文件在同一界面交叉比对，一次性确认所有字段—P1

5.6 多语言支持

能力说明优先级
OCR 语种中文、英文（P0），日文、韩文（P1），阿拉伯文、西班牙文、法文（P2）分阶段
界面语言简体中文（默认）、English（P1），可在用户设置中切换P1
智能语种检测上传时自动识别文件语种，匹配对应 OCR 模型，无需手动选择P0
小语种降级不支持的语种自动标记为"需人工录入"，不强制 OCR 避免产生错误数据P0

User Roles & Permissions

系统采用 RBAC（基于角色的访问控制）模型，支持多层级权限管理，确保数据安全和业务合规。

6.1 角色定义

角色典型岗位核心权限数据范围
超级管理员IT负责人 / 总经理全部功能 + 系统配置 + 用户管理 + 审计日志全公司
业务主管外贸经理 / 部门主管文件管理 + 审批 + 报表导出 + AI问答本部门
业务员外贸业务员 / 跟单员上传文件 + 查看/编辑自己的文件 + AI问答本人负责的客户/订单

财务人员财务 / 出纳查看银行水单/发票 + 导出财务数据 + 收付款核销财务类文件
只读访客审计 / 外部合作方仅查看指定文件，不可编辑/下载/导出按需授权

6.3 数据隔离策略

部门级隔离
不同业务部门的文件默认互不可见，业务主管仅能查看本部门数据。跨部门共享需管理员授权。
客户级隔离
业务员仅能查看自己负责的客户/订单相关文件，避免业务员之间交叉查看客户信息。
文件类型隔离
财务人员仅能访问银行水单、发票等财务类文件，不可查看合同条款细节等商务敏感信息。
临时授权机制
支持管理员对只读访客授予限时、限范围的文件访问权限，到期自动回收，适用于审计场景。

审批与工作流
Approval & Workflow

外贸单据涉及合规和财务准确性，系统内建文件全生命周期状态流转和多级审批机制。

7.1 文件状态流转

1
草稿
已上传未提交
2
待OCR
提交后自动识别
3
待核对
低置信度字段待确认
4
待审核
核对完成待主管审批
5
已归档
审核通过入库
6
已作废
标记无效保留记录

7.2 审批规则配置

审批场景触发条件审批层级超时策略
普通文件归档OCR核对完成后自动提交业务主管一级审批24h未审批自动提醒
金额超限文件单笔金额 ≥ USD 50,000业务主管 + 总经理二级审批48h未审批升级通知
字段修改已归档文件的字段被修改业务主管审批修改前后对比展示
文件删除/作废任何文件标记作废业务主管 + 管理员审批保留完整操作记录
灵活配置：审批规则支持管理员在系统设置中自定义，可按文件类型、金额阈值、部门分别配置不同的审批流程，无需开发介入。

7.3 人工核对任务分配

自动派单
OCR置信度低于阈值的字段自动生成核对任务，按文件类型匹配对应单证员，均衡分配工作量。
任务队列
单证员登录后看到待核对任务列表，按优先级（紧急程度、文件金额、等待时长）排序。
超时转派
核对任务超过4小时未处理，自动转派给同组其他单证员或升级通知业务主管。
批量核对
同一票货物的多个文件支持批量核对模式，在同一界面完成发票+箱单+提单的交叉比对。

业务大模型层
Business LLM Layer

清洗后的结构化数据持续投喂企业私有业务大模型，逐步构建外贸领域专属 AI 能力：

💬
智能文件问答
自然语言查询任意订单、金额、客户、HS编码等信息。例如："2024年上半年美国客户总出口金额是多少？"
🔔
跨文件异常检测
自动比对发票 vs 报关单金额、箱单 vs 提单数量，发现不一致自动告警，支持批量扫描历史文件。

🔁

新文件数据经清洗后定期（每周/每月）追加投喂，大模型对外贸合规、HS归类、汇率风险等理解持续提升。
数据安全说明：业务大模型部署于企业私有环境（私有云或本地服务器），所有文件数据不出境、不上传第三方平台，满足数据合规要求。

12.2 数据安全与合规

安全措施说明
传输加密全站 HTTPS / TLS 1.3，API 接口强制 TLS
存储加密文件存储采用 AES-256 加密，密钥由 KMS 管理
文件水印在线预览和导出 PDF 时自动叠加用户名+时间水印，防止截屏外泄
下载权限原件下载需具备对应角色权限，可按文件类型单独管控
登录安全支持企业 SSO（LDAP/SAML）、双因素认证（短信/OTP），登录失败锁定策略
异地容灾核心数据异地备份（同城双活或异地冷备），确保灾难场景可恢复
数据不出境所有文件和模型数据存储于境内服务器，满足数据出境合规要求

12.3 技术架构（PHP + Python 双语言）

架构原则：PHP 负责业务逻辑层（用户交互、权限、审批、文件管理），Python 负责数据与 AI 层（OCR、NLP、数据清洗、模型训练与推理）。两者通过 HTTP API + Redis 队列通信，共享 MySQL 数据库。部署于公司内部服务器。
层级组件技术选型说明

12.4 服务器部署方案

方案A：单服务器（初期推荐）
CPU: 8 核 / 内存: 16 GB / 硬盘: 1TB SSD
一台跑全部服务：Nginx + PHP + Python + MySQL + Redis

方案B：双服务器（业务量增长后）

适用场景：日均上传 > 200 份 或接入大模型推理
