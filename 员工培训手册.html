第一章 · 认识系统

1.1 系统是做什么的

贸易云档（TradeDoc） 是一套面向外贸企业的文件智能管理系统。它解决一个核心痛点：

报关单、商业发票、装箱单、银行水单等单证纸质 / PDF 散落各处，查找耗时，数据无法汇总。

系统通过 OCR + AI 自动识别单证上的关键字段（报关单号、金额、客户、HS 编码等），把"纸面信息"变成"可查询、可统计、可导出"的结构化数据。

系统能帮你做什么

角色你能用它做
业务员上传客户单证、关联订单、追踪进度、月度报表
单证员核对 OCR 识别结果、修正字段、确保数据准确
财务人员查发票/水单、按客户/订单汇总收支、导出财务台账
业务主管看部门整体单证完成率、审批关键文件、决策分析
管理员建账号、配权限、做系统维护

1.2 登录与首次密码修改

  1. 打开浏览器（推荐 Chrome / Edge），地址栏输入：http://公司服务器IP:8000。具体地址由 IT 或管理员提供。
  2. 输入账号密码。
    • 首次启用：默认管理员账号 admin / admin123
    • 普通员工：由管理员在系统设置中创建账号后发给你
  3. 点击「登录」，成功后进入仪表盘（Dashboard）。
  4. ⚠️ 重要 立即修改默认密码！路径：右上角头像 → 「修改密码」（或前往「系统设置 → 用户管理」）。
密码安全要求：
  • 不少于 8 位
  • 建议含字母 + 数字
  • 不要使用 123456、admin、姓名拼音等弱密码
  • 不要在多人共用电脑保存密码

左侧菜单（导航栏）：所有功能入口，按角色显示不同菜单项

1.4 角色与权限速查

超级管理员super_admin
全部 18 项权限；可建账号、改权限、看所有数据。一般由 IT 或老板担任。
业务主管manager
部门文件管理、审批、报表导出（12 项权限）。看本部门所有业务员的数据。
业务员salesman
上传与管理自己负责客户的单证（8 项）。日常上传的主力。
单证员doc_clerk
OCR 核对、字段修正、文件关联（6 项）。把"机器识别"变成"金标准数据"的关键角色。
财务人员finance
查看银行水单 / 发票、导出财务数据（7 项）。月底对账、做账时使用。
只读访客readonly
仅查看授权范围内文件（1 项）。一般给外部审计、临时合作方使用。

第二章 · 账号管理（管理员必读）

本章仅适用于 超级管理员。普通员工请跳到第三章。

2.1 创建新员工账号

  1. 左侧菜单 → 「系统设置」→ 切换到「用户管理」标签页。
  2. 点击右上角「+ 新建用户」按钮。
  3. 填写表单：
    • 用户名（登录用，必填，不可重复）：建议用员工姓名拼音，如 zhangsan
    • 真实姓名（必填）：用于显示，如 "张三"
    • 初始密码（必填，至少 6 位）：建议设为 员工名拼音+生日后4位，员工首次登录后强制改
    • 邮箱（选填，但建议填，用于密码找回）
    • 手机（选填）
    • 部门（选填）
    • 角色（必填）：从下拉选择，参考 1.4 节
    • 状态：启用 / 停用
  4. 点击「保存」。系统会立即创建账号。
  5. 把用户名 + 初始密码通过 企业微信 / 钉钉私聊 发给员工（不要群发！）。
💡 用户名规范建议：统一用全小写英文字母 + 数字，不含中文或特殊字符。这样未来导出、对账、API 调用都更稳定。

2.2 修改与停用账号

修改信息

  1. 「系统设置 → 用户管理」找到目标用户，点击「编辑」。
  2. 可修改：真实姓名、邮箱、手机、部门、角色、状态。
  3. 注意 用户名不能修改。如果用户名错了，需要新建一个账号、停用旧账号。

重置密码

  1. 找到目标用户 → 点击「重置密码」按钮。
  2. 系统生成新密码并显示一次（请截图或复制保存）。
  3. 把新密码私聊发给员工，提醒登录后立即修改。

员工离职处理

  1. 找到该员工账号 → 把「状态」改为停用。不要直接删除。
  2. 停用后该账号无法登录，但历史上传的文件、操作记录仍保留。
  3. 如果该员工负责的客户需要转给其他业务员，去「客户管理」改"负责人"字段。
不要删除已上传过文件的账号！删除会导致历史文件的"上传人"显示为空，影响审计和追溯。永远只停用。

2.3 角色分配建议

实际岗位建议角色理由

专职单证员doc_clerk专注核对，不需要管客户/订单
业务员 + 单证salesman + doc_clerk 双角色同时具备两类权限

第三章 · 基础数据维护

上传单证之前，必须先把"客户"和"订单"录入系统。这样后续上传的文件才能正确归类。本章建议管理员或业务主管在系统初始化时统一录入。

3.1 客户与供应商管理

客户 vs 供应商的区别

外贸业务有两条线：

采购 ← 供应商 · 销售 → 客户

系统的"客户管理"实际上是客户 + 供应商合一的，靠"类型"字段区分：

类型含义典型文件
customer 客户境外买家（销售出口）商业发票、装箱单、提单、原产地证
supplier 供应商境内/境外卖家（采购进口）采购合同、形式发票、银行水单
both 互为客户和供应商少见，既买又卖双方向单证

录入新客户/供应商

  1. 左侧菜单 → 「客户管理」→ 右上角「+ 新建」。
  2. 填写关键字段：
    • 类型：客户 / 供应商 / 互为
    • 公司名称（中文）
    • 英文名称（境外公司必填）
    • 简称（可选，方便快速识别）
    • 国家
    • 联系人 / 电话 / 邮箱
    • 地址
    • 开户行 / 银行账号 / SWIFT（涉及汇款时必填）
    • 税号
  3. 保存。客户即可在订单和上传页面被选中。
💡 录入顺序建议：第一次启用系统，建议先把常合作的前 20 个客户录入。其他客户在第一次发生业务时再补建即可，不必一次性录入数百个。

3.2 订单管理

什么时候建订单

外贸的常见流程：询盘 → 报价 → 接单 → 生产 → 出运 → 收款。建议在"接单"环节就把订单录入系统，后续所有单证（合同、发票、装箱单、报关单、提单……）都关联到这个订单，形成"一票货物档案"。

新建订单步骤

  1. 左侧菜单 → 「订单管理」→ 「+ 新建订单」。
  2. 填写：
    • 订单号：自定义，如 SO-2026-0088（销售订单）或 PO-2026-0010（采购订单）
    • 订单类型：出口（export） / 进口（import）
    • 客户：从下拉选（先在 3.1 录入）
    • 订单金额、币种
    • 贸易条款：FOB / CIF / EXW 等
    • 付款条款：如 T/T 30%订金，70%见提单
    • 目的国、装货港、卸货港
    • 负责业务员
  3. 保存。订单状态默认为 draft（草稿）。
  4. 业务推进时手动更新状态：draft → confirmed → shipping → completed。

3.3 文件类型说明（9 类）

系统预置 9 类外贸单证。上传时必须选择正确的文件类型，因为：

  • 不同类型的字段提取规则不同（报关单提取 HS 编码，发票提取金额）
  • 文件编号前缀不同（D = 报关单，INV = 发票...）
  • 归档目录按类型分
编号前缀中文名英文名典型出现场景
D报关单Customs Declaration清关阶段，海关申报
INV商业发票Commercial Invoice出口给客户、清关、收款依据
PL装箱单Packing List装柜、清关
BK银行水单Bank Receipt收/付汇凭证
BL提单/运单Bill of Lading海运/空运凭证
CO原产地证Certificate of Origin关税优惠、合规
CT合同协议Contract接单时签订
LC信用证Letter of Credit银行担保支付
DOC其他文件Other Documents授权书、检测报告等
文件编号规则：{前缀}{YYMMDD}-{4位序号}
例：D260428-0001 = 2026 年 4 月 28 日第 1 张报关单

第四章 · 核心业务流程

整体流程图

上传 → OCR 识别 → 字段提取 → 人工核对 → 审批 → 归档 → 导出

4.1 上传单证

单文件上传

  1. 左侧菜单 → 「上传录入」。
  2. 选择文件类型（9 类之一，必选）。
  3. 选择关联客户（强烈建议选，方便后续检索）。
  4. 选择关联订单（选填，但有订单的话务必关联）。
  5. 把文件 拖拽到上传区，或点击「选择文件」浏览本地。
    • 支持格式：PDF、JPG、PNG、TIFF、Word
    • 单文件大小：≤ 50MB
  6. 点击「开始上传」按钮。
  7. 系统立刻保存文件并自动调用 OCR/文本提取引擎，通常 5-30 秒完成识别。
  8. 识别完成后，文件出现在「文件管理」列表中，状态为 pending_review（待核对）或 pending_approval（待审批）。

批量上传

  1. 同样进入「上传录入」页面。
  2. 切换到「批量上传」标签。
  3. 选好文件类型 / 订单后，一次拖入多个文件（最多 50 个）。
  4. 系统会逐个上传并生成识别任务，进度条实时显示。
⚠️ 上传注意事项：
  • 批量上传时所有文件必须是同一类型（如全是装箱单）
  • 扫描件请选清晰扫描，模糊照片识别准确率会大幅下降
  • PDF 优于图片，电子版 PDF 优于扫描 PDF
  • 多页文档建议合并为一个 PDF，不要分散上传

4.2 查看与管理文件

文件列表页（/documents）

  1. 左侧菜单 → 「文件管理」。
  2. 顶部筛选区支持多维筛选：
    • 文件类型（报关单/发票/...）
    • 状态（草稿/待核对/待审批/已归档/作废）
    • 上传日期范围
    • 关联客户
    • 关联订单
    • 关键词（搜文件名、文档编号）
  3. 点击文件列任意位置进入详情页。

文件详情页

详情页左侧是文件预览（PDF / 图片），右侧是识别字段。常用操作：

操作怎么做用途
预览详情页默认显示对照原文核查
下载点「下载原文件」本地保存
编辑字段双击字段值修正 OCR 错误
新增字段点「+ 新增字段」补充模板外的字段
删除字段字段行右侧 🗑误识别的字段
重新识别点「重新 OCR」识别质量差时使用
导出 Excel点「导出 Excel」转交财务/报表
变更状态点状态下拉归档 / 作废

4.3 人工核对（单证员）

OCR 自动识别后，置信度低于 80% 的字段会生成"核对任务"，由单证员逐一确认。这一步是数据准确性的关键防线。

核对操作步骤

  1. 左侧菜单 → 「人工核对」。
  2. 看到任务列表，按优先级（紧急/普通）和时间排序。
  3. 点击任务进入核对页：
    • 左：文件原图/PDF（带字段位置高亮，俗称 bbox 标注）
    • 右：OCR 识别出的字段值，标红的是需要确认的
  4. 逐字段对照原图核对：
    • 如果识别正确 → 点「✓ 确认」
    • 如果识别错误 → 直接在字段框里修改 → 「✓ 确认」
    • 如果该字段在原文中根本没有 → 清空值 → 「✓ 确认」
    • 如果自己无法判断 → 「跳过」转交他人
  5. 所有字段确认后，任务自动关闭，文件状态推进到 pending_approval（待审批）。
💡 核对效率技巧：
  • 金额类字段：先看币种，再看数字，注意小数点位置
  • 日期类字段：注意年月日顺序（中式 vs 美式）
  • HS 编码：必须是 8-10 位数字，不能有字母
  • 客户名称：核对英文拼写，特别是首字母大小写

4.4 字段编辑与重新识别

什么时候手动编辑字段

  • OCR 漏识别了某个字段 → 「+ 新增字段」
  • OCR 识别错了几个字符 → 双击字段值修改
  • OCR 误识别了一个不该有的字段 → 点 🗑 删除

什么时候触发重新 OCR

  • 识别质量普遍很差（多个字段都错） → 重新 OCR 整张
  • 上传的文件被替换或修改了 → 重新 OCR
  • 系统升级后想用新引擎重跑 → 重新 OCR
注意：重新 OCR 会覆盖未经人工确认的字段值，但已经确认过的字段（带"已核对"标志）不会被覆盖。

4.5 文件状态流转

了解文件状态有助于追踪进度。

draft (草稿)
   ↓ 上传后自动转
ocr_processing (识别中)
   ↓ 引擎处理完成
pending_review (待核对)  ← 有低置信度字段
   ↓ 单证员确认
pending_approval (待审批)
   ↓ 主管点"通过"
approved (已审批)
   ↓ 自动归档
archived (已归档)

任意状态都可以 → voided (作废)
状态含义谁能操作
draft刚上传，OCR 未启动系统自动
ocr_processing识别中，请等待系统自动
pending_review等待单证员核对单证员
pending_approval等待主管审批业务主管
approved已通过系统自动归档
archived已归档，正常使用中不可改
voided已作废管理员可改回

第五章 · 数据应用

5.1 导出 Excel

单文件导出

  1. 进入文件详情页 → 右上角「导出 Excel」按钮。
  2. 浏览器自动下载，文件名格式：{文档编号}_{文件类型}.xlsx。
  3. Excel 内含：基本信息、所有识别字段、置信度、核对状态。

批量导出

  1. 在「文件管理」列表页用筛选器选好范围（如：本月所有报关单、客户 A 的所有发票）。
  2. 列表勾选要导出的文件（或点表头复选框全选当前页）。
  3. 点顶部「批量导出」按钮。
  4. 稍等几秒，浏览器自动下载一个 Excel，包含所有勾选文件的字段数据，按类型分 Sheet。
💡 应用场景：
  • 月末：导出本月所有商业发票，给财务做账
  • 季报：导出本季出口报关单，做出口额统计
  • 审计：按客户导出所有单证给审计

5.2 数据报表

  1. 左侧菜单 → 「数据报表」。
  2. 选择报表维度：按文件类型 / 按客户 / 按月份 / 按业务员。
  3. 选择时间范围。
  4. 系统实时计算并展示图表（柱状图、饼图、趋势线）。
  5. 点「导出图表」可保存为图片，点「导出数据」可下载 Excel。

5.3 AI 智能问答

系统集成了基于业务数据的 AI 问答，可以用大白话直接查询，无需写 SQL 或导 Excel。

  1. 左侧菜单 → 「AI 问答」。
  2. 在输入框输入问题（中文/英文都行）。
  3. 点击「发送」，等几秒，AI 自动查询业务数据库并回答。

  • 指明实体名称：客户名、订单号、文件类型
  • 多轮对话：可以接着上轮回答继续追问，AI 记得上下文
  • 不确定时：用"列出所有..."、"统计..."、"对比..."这类动词开头

6.1 系统通知

通知操作

  1. 点击 🔔 图标展开通知列表。
  2. 点击单条通知 → 自动跳转到对应文件或任务页。
  3. 「全部已读」一键清空小红点。

6.2 多人协作场景

场景一：业务员出差，同事代为上传

  1. 同事用自己的账号登录，进入「上传录入」。
  2. 上传时务必选择正确的客户和订单，这样文件归属客户，不归属上传人。
  3. 原业务员回来后可在「文件管理」筛选自己负责的客户，仍能看到这些文件。

场景二：单证员请假，其他人帮忙核对

  1. 临时单证员账号有 review.execute 权限即可。
  2. 在「人工核对」看到的任务列表是所有未核对任务，不区分原指派人。
  3. 核对完成后系统自动记录"实际核对人"，方便事后追溯。

场景三：业务交接

  1. 管理员登录 → 「客户管理」找到要交接的客户。
  2. 把「负责业务员」字段从离职人改为接替人。
  3. 历史文件归属不变（仍记录原上传人），但新文件归到新业务员。

第七章 · 端到端实战案例

案例一

业务员小张上传一份新客户的报关单

背景：小张刚接了一笔出口订单，给美国客户 "ABC Trading" 出口集成电路芯片。今天拿到了海关报关单 PDF，需要录入系统。

操作步骤

  1. 登录系统。用账号 zhangsan 登录。
  2. 建客户（首次合作）。
    • 左侧菜单 →「客户管理」→「+ 新建」
    • 类型：客户
    • 公司名称（中文）：ABC 贸易
    • 英文名称：ABC Trading Co., Ltd.
    • 国家：美国 USA
    • 联系人 / 电话 / 邮箱：填齐
    • 保存
  3. 建订单。
    • 「订单管理」→「+ 新建订单」
    • 订单号：SO-2026-0088
    • 类型：出口
    • 客户：选刚建好的 ABC 贸易
    • 订单金额：48200 USD
    • 贸易条款：FOB 上海
    • 目的国：美国
    • 装货港 / 卸货港 / 负责业务员（自己）
    • 保存
  4. 上传报关单。
    • 「上传录入」
    • 文件类型：报关单
    • 关联订单：选 SO-2026-0088（系统会自动带出客户 ABC 贸易）
    • 拖入 PDF → 「开始上传」
  5. 等待 OCR（约 10-30 秒）。系统识别完后，文件出现在「文件管理」列表，状态多半为「待核对」（因为是新模板，可能有几个字段置信度低）。
  6. 查看识别结果。点开文件详情，浏览左侧 PDF 和右侧字段。
  7. 等单证员核对。带"待核对"状态的文件会自动出现在单证员的工作台上。
💡 小张避坑提示：
  • 客户公司名一定要双语录入，英文名出错会影响所有英文单证的核对
  • 订单金额要和报关单金额一致，便于后期对账
案例二

单证员小李核对低置信度字段

背景：小李是单证员，每天上班第一件事就是清理积压的核对任务。

  1. 登录账号 lili。仪表盘上看到"未核对任务：12"。
  2. 进入「人工核对」页。按"创建时间"升序排序，从最旧的开始处理。
  3. 打开第一个任务。
    • 左侧显示原 PDF：一张报关单
    • 右侧显示 OCR 识别的 10 个字段，其中 3 个红色 = 待确认
  4. 核对字段。
    • "贸易方式" 识别为 "1分贸易"（应该是 "一般贸易"）→ 修改为 "一般贸易" → ✓
    • "HS 编码" 识别为 "8542310OO1"（O 是字母，应该是数字 0）→ 改为 "8542310001" → ✓
    • "运输方式" 留空（原图也没有）→ 保持空 → ✓
  5. 提交。三个字段都确认后，任务自动关闭，文件转为「待审批」。
  6. 继续下一个任务。循环上面流程，直到任务列表清空。
⚠️ 小李工作要点：
  • 不能"看着差不多就点确认"，必须每个字段对照原文核对
  • 无法判断的字段（比如客户的特殊地址缩写）→ 点「跳过」，让业务员或主管处理
  • 修改后的内容会被记录到 field_change_logs，作为系统训练数据
案例三

财务月底导出当月发票汇总

背景：财务王姐每月 1 号要给老板交一份"上月销售发票汇总 Excel"。

  1. 登录账号 wangli（角色 finance）。
  2. 进入「文件管理」。
  3. 设置筛选条件。
    • 文件类型：商业发票
    • 上传日期：2026-04-01 至 2026-04-30
    • 状态：已归档（排除掉未审批和草稿）
  4. 查看结果。列表显示本月 47 张已归档发票。
  5. 全选 → 批量导出。点列表表头复选框全选 → 点「批量导出 Excel」。
  6. 打开 Excel。文件含：发票号、日期、买家、金额、币种、付款条款 等字段。
  7. 加工成汇总表。在 Excel 里用数据透视表按客户汇总，加几列同比环比。
  8. 发给老板。
💡 王姐进阶用法：
  • 可以同时打开「AI 问答」问："本月发票总金额按客户分组是多少？"——直接得到结构化数据，省略一步透视表

第八章 · 常见问题 FAQ

8.1 登录与账号

Q：忘记密码了怎么办？
A：联系管理员重置。管理员路径：「系统设置 → 用户管理」→ 找到你 → 「重置密码」→ 把新密码发给你。
Q：能用同一账号在多台电脑登录吗？
A：可以。但同一时间多端登录可能导致数据冲突，建议保持单端使用。
Q：用户名能改吗？
A：不能。如果用户名错了，需要管理员新建一个账号、停用旧账号。
Q：登录后看不到某个菜单？
A：菜单按角色显示。比如只读角色看不到"上传录入"和"系统设置"。如有疑问联系管理员检查角色配置。

8.2 上传与识别

Q：上传后一直显示"识别中"，怎么办？
A：先等 30 秒。若超过 2 分钟仍未完成，可能是 OCR 服务未启动。联系管理员检查 Python OCR 服务（start.bat）。
Q：识别结果几乎全错怎么办？
A：
  1. 检查文件是否清晰（模糊扫描件识别效果差）
  2. 确认文件类型选对了（报关单选成发票，识别字段就会错位）
  3. 详情页点「重新 OCR」再跑一次
  4. 仍然不行 → 联系单证员手动录入
Q：批量上传时有几个文件失败了？
A：失败的文件会单独提示。点击错误信息可以看具体原因（格式不支持、文件过大、网络异常等）。修正后单独重传即可。
Q：能上传 Word 文档吗？
A：可以，系统会用 python-docx 直接提取文字，比 OCR 还准。但建议合同协议这类才用 Word，报关单/发票仍用 PDF。

8.3 字段与导出

Q：识别字段不全，缺少几个我想要的？
A：详情页点「+ 新增字段」手动添加。如果该字段是所有文件都缺，说明字段模板没配置，联系管理员去「系统设置 → 字段模板」补一个。
Q：导出 Excel 后有些字段是空的？
A：说明这些字段在原文件里没有识别到或没人工补录。建议：
  1. 导出前先把待核对的文件处理完
  2. 列表页筛选时排除掉草稿和未审批的
Q：能定制导出模板吗（比如老板想要他自己的格式）？
A：当前版本提供标准 Excel 模板。定制模板属于二次开发，请提需求给 IT。
