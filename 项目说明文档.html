外贸文件智能管理系统 — 功能与使用说明

v2 · 文档更新于 2026-04-28

一、项目概述

贸易云档是一套面向外贸企业的文件智能管理系统,通过 OCR + AI 自动提取报关单、商业发票、装箱单、银行水单等 外贸单证的关键字段,实现文件从上传 → 识别 → 核对 → 归档 → 导出 的全流程数字化管理。

📤

智能上传

支持单文件与批量上传,自动生成业务编号,按年月分目录归档。

🔍

OCR 识别

阿里云通义千问 + 本地模型双引擎,自动提取关键字段。

✅

人工核对

低置信度字段自动转单证员核对,确保数据准确。

📊

报表导出

支持单文件/批量导出 Excel,自定义字段模板。

🤖

AI 问答

基于业务数据的智能问答,辅助决策分析。

🔐

权限管控

基于角色的细粒度权限,6 大角色 18 项权限。

五、业务流程

文件处理主流程

上传 → OCR 识别 → 字段提取 → 人工核对 → 审批 → 归档 → 导出

详细步骤

  1. 上传:用户通过 /upload 页面上传文件(支持单文件/批量),系统生成文件编号,存储到 storage/app/private/documents/
  2. OCR 识别:PHP 后端调用 Python FastAPI 服务进行 OCR;引擎按优先级:阿里云 DashScope > 本地模型 > Mock
  3. 字段提取:OCR 完成后根据字段模板自动提取关键字段(如报关单号、金额、日期等),存入 document_fields 表
  4. 人工核对:置信度低于阈值(默认 80%)的字段自动生成核对任务,由单证员在 /review 页面逐一确认或修正
  5. 审批:变更状态触发审批规则,支持多级审批
  6. 归档:审批通过后文件状态变为 archived,记录归档时间
  7. 导出:支持单文件/批量导出 Excel

文件状态流转

draft → ocr_processing → pending_review → pending_approval → approved → archived
                                                                    ↓
                                                                voided (任意状态可作废)

六、使用说明

6.1 登录系统

  1. 打开浏览器访问:http://服务器IP:8000
  2. 输入默认管理员账号:admin / admin123
  3. 点击 "登录" 进入仪表盘
  4. 重要 首次登录后,请立即在「系统设置 → 用户管理」修改默认密码

6.2 上传文件

  1. 点击左侧菜单「上传录入」进入 /upload 页面
  2. 选择文件类型(报关单 / 发票 / 装箱单 / 银行水单 等 9 种)
  3. 选择关联订单和客户(可选,用于自动归类)
  4. 拖拽文件或点击上传按钮(支持 PDF / 图片,单文件 ≤100MB)
  5. 系统自动调用 OCR 引擎识别,稍候片刻即可在「文件管理」中查看
批量上传:可同时拖入多个文件,系统会逐个处理并生成任务。 OCR 识别耗时与文件页数和清晰度有关,通常每文件 5-30 秒。

6.3 管理文件

  1. 进入「文件管理」/documents 页面
  2. 使用顶部筛选器按文件类型、状态、日期、客户、订单等条件筛选
  3. 点击文件名进入详情页,可:
    • 预览原始文件(支持 PDF / 图片)
    • 查看 OCR 识别出的字段值
    • 编辑字段(双击单元格)
    • 新增/删除自定义字段
    • 下载原文件
    • 导出 Excel
    • 触发重新 OCR(识别有误时使用)
  4. 批量操作:列表页勾选多条 → 顶栏按钮(批量导出/批量删除)

6.4 人工核对(单证员)

  1. 登录单证员账号,进入「人工核对」/review 页面
  2. 查看待核对任务列表(按优先级/时间排序)
  3. 点击任务进入核对页:左侧显示原文件,右侧显示 OCR 识别字段
  4. 逐字段比对原文,修正错误内容
  5. 点击「确认」完成核对,或点击「跳过」转交他人

6.5 导出数据

  1. 单文件导出:文件详情页点击「导出 Excel」按钮
  2. 批量导出:列表页筛选所需文件 → 勾选 → 点击「批量导出」
  3. 报表导出:进入「数据报表」/reports 页面,选择维度后导出

6.6 AI 智能问答

  1. 进入「AI 问答」/ai 页面
  2. 用自然语言提问,例如:
    • "本月报关单总金额是多少?"
    • "客户张三本季度的发票有几张?"
    • "哪些文件还在待核对状态?"
  3. AI 会基于系统业务数据自动查询并回答

部署步骤

  1. 服务器准备:安装 phpStudy Pro;安装 VC++ 2022 运行库;解压 PHP 8.5.5 NTS 到 phpStudy 扩展目录
  2. 复制项目:整个 business_file_management 目录复制到目标服务器(含 vendor),推荐路径 C:\product\business_file_management\
  3. 配置 .env:修改 tradedoc/.env: 
    APP_ENV=production
APP_DEBUG=false
APP_URL=http://服务器IP:8000
DB_PASSWORD=实际密码
  4. 建库 + 导入数据:在 phpStudy MySQL 里建库并依次执行:
    • database/install.sql(建表 + 初始数据)
    • database/fix_missing_tables.sql(Laravel 内置表)
    • database/fix_spatie_tables.sql(Spatie 权限表 + admin 绑定)
  5. 缓存 Laravel 配置: 
    cd 项目路径\tradedoc
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan storage:link
  6. 配置 Nginx 站点:phpStudy 面板 → 创建网站,端口 8000,根目录指向 tradedoc/public;替换 vhost.conf 为 deploy/nginx_tradedoc.conf 内容
  7. 开 Windows 防火墙: 
    New-NetFirewallRule -DisplayName "TradeDoc HTTP 8000" `
  -Direction Inbound -Protocol TCP -LocalPort 8000 `
  -Action Allow -Profile Any
  8. 访问 + 登录:浏览器打开 http://服务器IP:8000,用 admin/admin123 登录

常见问题 FAQ

Q1: 502 Bad Gateway?
原因:phpStudy 的 PHP-CGI 池死了。
解决:phpStudy 面板重启 PHP;查 tradedoc_error.log 是否 9001 连接被拒。
Q2: 登录后 dashboard 报错?
原因:缓存的 config 跟实际不一致。
解决:php artisan config:clear && php artisan config:cache
Q3: 修改代码不生效?
Laravel 的 view/route/config 缓存需要清掉重建。如果只改了一个 PHP 文件, OPcache 默认会自动重新编译(opcache.validate_timestamps=1)。
Q4: 上传文件大小超限?
改 php.ini 的 upload_max_filesize 和 post_max_size; 改 Nginx 配置的 client_max_body_size;重启 PHP 和 Nginx。
Q5: 局域网内其他电脑访问不到?
1. 检查 Nginx 是否监听 0.0.0.0(不是 127.0.0.1)
2. 检查 Windows 防火墙是否开了 8000 端口
3. 检查公司网络是否有 VLAN 隔离
Q6: MySQL 连不上?
检查 phpStudy MySQL 是否启动;检查 .env 里 DB_PASSWORD 是否正确; 跑 php artisan db:show 验证连接。
