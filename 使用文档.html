1. 系统概述

贸易云档（TradeDoc） 是一套面向外贸企业的文件智能管理系统，通过 OCR + AI 自动提取报关单、商业发票、装箱单、银行水单等外贸单证的关键字段，实现单证从上传、识别、核对到归档、导出的全流程数字化。

核心业务流程

上传 → OCR 识别 → 字段提取 → 人工核对 → 审批 → 归档 → 导出

支持的单证类型（9 类）

类型编号前缀类型编号前缀
报关单D提单/运单BL
商业发票INV原产地证CO
装箱单PL合同协议CT
银行水单BK信用证LC
其他文件DOC

文件编号格式：{前缀}{YYMMDD}-{4位序号}，例：D260428-0001

2. 登录与界面

2.1 登录系统

  1. 打开浏览器（推荐 Chrome / Edge），访问 http://服务器IP:8000。
  2. 输入用户名和密码。
    • 默认超级管理员：admin / admin123（部署后请立即修改）
    • 普通员工账号由管理员创建后发放
  3. 点击「登录」，成功后进入仪表盘。

2.2 修改密码

  1. 点击右上角头像 → 「修改密码」。
  2. 输入旧密码、新密码、确认新密码。
  3. 提交后立即生效，下次登录使用新密码。
首次登录默认账号 admin 后请立即修改密码。 密码建议至少 8 位，包含字母和数字。

2.3 界面布局

登录后界面分三块：

  • 左侧菜单：所有功能入口，按角色显示不同菜单项
  • 顶部栏：通知图标 🔔、当前用户头像、退出按钮
  • 主区域：根据点击的菜单显示对应内容

菜单一览

菜单路径用途
📊 仪表盘/dashboard本月统计、待办提醒
📤 上传录入/upload上传单证
📁 文件管理/documents查询/筛选/编辑
✅ 人工核对/review核对 OCR 结果
🛒 订单管理/orders维护订单

📈 数据报表/reports统计分析

⚙️ 系统设置/settings仅管理员可见

2.4 退出登录

点击右上角头像 → 「退出登录」。系统会清除当前 Token，回到登录页。

3. 基础数据

上传单证之前，先把客户和订单维护好。这样后续上传的文件才能正确归类、检索。

3.1 客户管理

系统的"客户管理"统一管理 客户和供应商，靠"类型"字段区分。

类型含义对应业务方向
客户 (customer)境外买家销售出口
供应商 (supplier)境内/境外卖家采购进口
互为 (both)既买又卖双方向

新建客户/供应商

  1. 左侧菜单 → 「客户管理」→ 「+ 新建」。
  2. 填写：
    • 类型（必填）：客户 / 供应商 / 互为
    • 公司名称（必填，中文）
    • 英文名称（境外公司必填）
    • 简称、国家、联系人、电话、邮箱、地址
    • 开户行 / 银行账号 / SWIFT / 税号（涉及汇款必填）
  3. 保存。客户即可在订单和上传页被选中。

查询、修改、删除

  • 查询：客户列表页用顶部搜索框按公司名/简称模糊查
  • 修改：列表点击客户名 → 「编辑」
  • 删除：有关联订单或文件的客户不能删除，只能停用

3.2 订单管理

建议在接单环节就录入订单，后续单证（合同、发票、装箱单、报关单、提单……）都关联到该订单，形成"一票货物档案"。

新建订单

  1. 左侧菜单 → 「订单管理」→ 「+ 新建订单」。
  2. 填写：
    • 订单号：自定义，如 SO-2026-0088
    • 订单类型：出口 (export) / 进口 (import)
    • 客户：从下拉选
    • 订单金额、币种、贸易条款（FOB/CIF/EXW）、付款条款
    • 目的国、装货港、卸货港
    • 负责业务员
  3. 保存。订单状态默认为 draft。

订单状态流转

draft (草稿) → confirmed (已确认) → shipping (运输中) → completed (已完成)
                                                            ↓
                                                       cancelled (取消)

3.3 文件类型

系统预置 9 类外贸单证（见第 1 章末尾的速查表）。上传时必须正确选择文件类型，因为：

  • 不同类型对应不同的字段提取规则
  • 不同类型的文件编号前缀不同
  • 归档目录按类型分

4. 上传单证

4.1 单文件上传

  1. 左侧菜单 → 「上传录入」。
  2. 选择 文件类型（9 类之一，必选）。
  3. 选择 关联客户（建议必填）。
  4. 选择 关联订单（选填，有订单务必关联）。
  5. 把文件拖拽到上传区，或点击「选择文件」浏览本地。
    • 支持格式：PDF、JPG、PNG、TIFF、Word (.docx)
    • 单文件大小：≤ 50MB
  6. 点击「开始上传」。系统保存文件并自动调用 OCR/文本提取引擎。
  7. 识别耗时通常 5-30 秒。完成后文件出现在「文件管理」中。

4.2 批量上传

  1. 「上传录入」→ 切换到「批量上传」标签。
  2. 选好文件类型 / 订单 / 客户。
  3. 一次拖入多个文件（最多 50 个）。
  4. 系统逐个上传并生成识别任务，进度条实时显示。
批量上传约束：所有文件必须是同一类型（如全是装箱单）； 同一客户/订单。如需混合上传，请逐个单独上传。

4.3 上传后处理

上传成功后，系统按以下顺序处理：

  1. 文件落盘：物理文件存到 storage/app/private/documents/年/月/类型/，原始文件名记录在数据库。
  2. 生成业务编号：按"前缀 + 日期 + 序号"自动生成，如 INV260428-0001。
  3. OCR/文本提取：
    • 电子版 PDF / Word → 直接提取文字（pdfplumber / python-docx）
    • 扫描件 / 图片 → 调用 OCR 引擎（阿里云通义千问 / 本地模型）
  4. 字段提取：根据文件类型自动提取关键字段。
  5. 状态判定：
    • 有置信度 < 80% 的字段 → 转 pending_review（待核对）
    • 全部高置信度 → 转 pending_approval（待审批）

5. 文件管理与查询

5.1 文件列表

左侧菜单 → 「文件管理」→ 进入文件列表页 /documents。

列表默认按上传时间倒序，每页 20 条。列：

  • 文档编号、原始文件名、文件类型
  • 关联客户、关联订单
  • 状态（草稿/识别中/待核对/待审批/已归档/作废）
  • 上传人、上传时间
  • 操作（查看 / 下载 / 删除）

5.2 筛选与搜索

列表页顶部筛选区支持多维条件叠加：

筛选项说明
文件类型报关单、发票、装箱单等 9 类
状态草稿/待核对/待审批/已归档/作废
日期范围按上传时间起止
关联客户下拉选客户/供应商
关联订单下拉选订单
关键词搜文档编号、原始文件名

多个条件同时生效（AND 关系）。点「重置」清空所有条件。

5.3 文件详情

列表点击文件名进入详情页 /documents/:id。

详情页布局：

  • 左侧：文件预览（PDF / 图片直接渲染，Word 转 PDF 显示）
  • 右侧：识别字段列表，每行显示字段名、字段值、置信度、操作按钮

详情页常用操作

操作按钮位置用途
下载原文件右上「下载」下载到本地，文件名为原始名
导出 Excel右上「导出」单文件导出为 Excel
重新 OCR右上「重新 OCR」识别质量差时使用
变更状态状态下拉归档 / 作废
查看 OCR 缓存底部「查看缓存」看 OCR 原始返回
删除文件右上「删除」软删除，可恢复

5.4 字段编辑

修改字段值

  1. 双击字段值单元格 → 进入编辑模式。
  2. 输入新值 → 回车 / 点击空白处保存。
  3. 修改后系统会记录变更日志（变更前/后值、修改人、时间）。

新增字段

  1. 详情页字段列表底部 → 点「+ 新增字段」。
  2. 选字段模板（系统预置）或自定义字段名。
  3. 填字段值 → 保存。

删除字段

字段行右侧 🗑 图标 → 确认删除。仅删除该文件的字段值，不影响模板。

已经"人工核对确认"的字段，重新 OCR 不会被覆盖。OCR 只会更新未确认的字段。

5.5 文件状态

draft (草稿)
   ↓ 上传后自动转
ocr_processing (识别中)
   ↓ 引擎处理完成
pending_review (待核对)        ← 有低置信度字段
   ↓ 单证员确认
pending_approval (待审批)
   ↓ 主管审批
approved (已审批)
   ↓ 自动归档
archived (已归档)

任意状态 → voided (作废)

6. 人工核对

OCR 识别后，置信度 < 80% 的字段会自动生成"核对任务"，由单证员逐一确认。这一步保证识别结果的准确性。

进入核对工作台

  1. 左侧菜单 → 「人工核对」/review。
  2. 页面分为两块：
    • 顶部：统计卡片（待核对数、今日已核对、近 7 日完成）
    • 下方：核对任务列表

核对单个任务

  1. 点击任务进入核对页：
    • 左侧：原文件预览（PDF/图片，字段位置带高亮标注）
    • 右侧：待核对字段列表（标红的需要确认）
  2. 逐字段比对原文：
    • 识别正确 → 点「✓ 确认」
    • 识别错误 → 直接修改字段值 → 「✓ 确认」
    • 该字段原文没有 → 清空值 → 「✓ 确认」
    • 无法判断 → 「跳过」转给他人
  3. 所有字段确认后，任务自动关闭，文件状态推进到「待审批」。
核对要点：
  • 金额：先确认币种，再核对数字与小数点
  • 日期：注意年月日顺序（中式 vs 美式）
  • HS 编码：8-10 位数字，不能含字母（容易把 O 看成 0）
  • 客户名：核对英文拼写和大小写

7. 数据导出与报表

7.1 单文件导出

  1. 进入文件详情页。
  2. 右上角 → 「导出 Excel」按钮。
  3. 浏览器自动下载，文件名格式 {文档编号}_{类型}.xlsx。

Excel 内容：基本信息、所有识别字段、置信度、核对状态。

7.2 批量导出

  1. 「文件管理」用筛选器选好范围（如本月所有报关单）。
  2. 列表勾选要导出的文件（或点表头复选框全选当前页）。
  3. 顶部「批量导出」按钮。
  4. 下载的 Excel 按文件类型分 Sheet，便于汇总分析。
批量导出建议先用筛选器收窄范围，一次导出不要超过 1000 条，否则可能超时。

7.3 数据报表

  1. 左侧菜单 → 「数据报表」/reports。
  2. 选择报表维度：按文件类型 / 按客户 / 按月份 / 按业务员。
  3. 选择时间范围。
  4. 系统实时计算并展示图表（柱状图、饼图、趋势线）。
  5. 「导出图表」保存为图片，「导出数据」下载 Excel。

常用报表

报表名用途
本月文件类型分布看业务结构
按客户的单证数量看大客户活跃度
本季 OCR 准确率评估系统效果
核对待办积压单证员工作量监控
出口总金额趋势业绩分析

8. AI 智能问答

用自然语言直接查询业务数据，不需要写 SQL 或导 Excel。

使用步骤

  1. 左侧菜单 → 「AI 问答」/ai。
  2. 在输入框输入问题。
  3. 点击「发送」或按回车键。
  4. 等几秒，AI 自动查询业务数据并回答。
  5. 支持多轮对话，可继续追问。

问题示例

你想知道这样问
本月报关单总额本月所有报关单的总成交金额是多少？
客户业务量客户 ABC Trading 今年的发票一共多少张？
积压情况现在还有几个文件在待核对状态？
员工业绩张三本月上传了多少份文件？
对比分析本季度出口总额比上季度涨了多少？
异常排查哪些发票的 OCR 置信度低于 70%？
提问技巧：
  • 指明时间范围："本月"、"上季度"、"2026 年 4 月"
  • 指明具体实体：客户名、订单号、文件类型
  • 用动词开头："列出..."、"统计..."、"对比..."
  • 多轮对话：AI 会记住上下文，可以连续追问

9. 系统设置（管理员）

本章仅限 超级管理员。普通员工跳过即可。

9.1 用户管理

新建用户

  1. 左侧菜单 → 「系统设置」→ 「用户管理」标签。
  2. 右上角「+ 新建用户」。
  3. 填写：
    • 用户名（登录用，必填，不可重复，创建后不可修改）
    • 真实姓名（必填）
    • 初始密码（至少 6 位）
    • 邮箱、手机、部门（选填）
    • 角色（必填，从下拉选）
    • 状态：启用 / 停用
  4. 保存。把用户名 + 初始密码私聊发给员工。

修改用户

  • 可修改：真实姓名、邮箱、手机、部门、角色、状态
  • 不可修改：用户名（若错需新建账号 + 停用旧账号）

重置密码

  1. 用户列表找到目标用户 → 「重置密码」。
  2. 系统生成新密码，显示一次（截图或复制）。
  3. 把新密码私聊给员工。

员工离职

不要删除离职员工账号！把"状态"改为停用即可。 删除会导致历史文件的上传人为空，破坏审计追溯。

9.3 字段模板

每种文件类型对应一套"字段模板"，决定 OCR 提取哪些字段。

  1. 「系统设置」→ 「字段模板」标签。
  2. 选择文件类型，查看当前模板字段。
  3. 可增删字段（需 super_admin 权限）。
  4. 修改后会影响后续新上传文件的识别，不影响历史已识别文件。

9.4 操作日志

「系统设置」→ 「操作日志」可查看：

  • 用户登录/退出记录
  • 文件上传/编辑/删除记录
  • 用户管理操作（建/改/停用）
  • 字段变更历史（谁、什么时候、改了什么）

日志按时间倒序，支持按用户、操作类型、时间范围筛选。日志不可删除。

10. 常见问题

登录与账号

Q：忘记密码？
联系管理员重置。「系统设置 → 用户管理」找到你 → 「重置密码」。
Q：登录后看不到某个菜单？
菜单按角色显示，如有疑问联系管理员检查角色配置。
Q：用户名能改吗？
不能。如果用户名错了，需要管理员新建账号、停用旧账号。

上传与识别

Q：上传后一直显示"识别中"？
先等 30 秒。超过 2 分钟未完成，可能 OCR 服务未启动，联系管理员检查 Python OCR 服务。
Q：识别结果几乎全错？
① 检查文件是否清晰；② 确认文件类型选对；③ 详情页点「重新 OCR」；④ 仍不行 → 手动录入。
Q：批量上传时有几个文件失败？
失败的文件单独提示原因（格式不支持、文件过大、网络异常）。修正后单独重传。
Q：能上传 Word 文档吗？
可以。系统对 .docx 直接用 python-docx 提取文字，比扫描 OCR 还准。

字段与导出

Q：识别字段不全，缺几个想要的？
详情页点「+ 新增字段」手动添加。所有文件都缺 → 联系管理员补字段模板。
Q：导出 Excel 后有些字段是空的？
说明这些字段在原文件中未识别到或未人工补录。建议导出前把待核对的处理完。
Q：能定制导出模板吗？
当前版本只支持标准模板。定制需求请提给 IT。
